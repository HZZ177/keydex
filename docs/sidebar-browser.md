# 侧边栏浏览器与网页批注

Keydex 的侧边栏浏览器是一块与会话、文件和审阅面板并列的原生浏览器面板。它使用 Tauri/WebView2 Surface 承载真实网页，不使用受站点 `X-Frame-Options` 和 CSP 限制的 iframe。

## 运行环境

- 正式安装包和 `pnpm run tauri:dev` 提供原生浏览器宿主。
- 单独运行 `pnpm run dev` 只启动 Vite 渲染层，无法在普通浏览器页面内嵌任意第三方站点；这不是受支持的浏览器 Surface 验收环境。
- 浏览器和网页批注在正式版默认启用。内部诊断探针在生产环境默认关闭。

开发时如需让 Tauri 壳复用 PyCharm 中运行的后端，可在启动壳的 PowerShell 中设置运行时变量：

```powershell
cd .\desktop
$env:KEYDEX_DEV_AGENT_BASE_URL = "http://127.0.0.1:8765"
pnpm run tauri:dev
```

该变量仅在 Debug 壳中生效。命中后不会启动或停止内置 sidecar；未设置时保持原有 sidecar 启动行为。如需使用 `tauri:dev:isolated`，必须先在仓库根目录的另一个终端运行 `pnpm run dev:frontend`，因为隔离配置不会自行启动 Vite。完整说明见项目 README 的“Tauri 壳连接已运行的 PyCharm 后端”。

## 浏览器能力

在右侧初始页新建“浏览器”面板后，可以使用：

- 多标签，以及前进、后退、刷新和停止加载。
- 地址输入与搜索、页内查找、页面缩放。
- 文件选择、下载进度与下载结果定位。
- 摄像头、麦克风、通知等网站权限的显式授权提示。
- 普通 Profile 与无痕 Profile。普通 Profile 可恢复安全的标签元数据；无痕标签不会进入持久化恢复状态。

浏览器 Surface 只在对应标签和侧边栏可见时显示。切换标签、收起侧边栏、切换任务或最小化窗口时，宿主会隐藏或降温 Surface。这是原生子窗口的生命周期管理，用来避免覆盖其他 Keydex 内容、截获输入和无效占用资源，不是产品功能开关。

## 结构化选择与批注

浏览页面时可按内容选择合适的定位方式：

- 文本：保存文本上下文、文本位置和必要的页面定位信息。
- 元素：保存元素语义、稳定属性和结构定位信息。
- 区域：保存页面坐标、视口和截图资产引用。

批注由内容、颜色、状态和结构化锚点组成。页面变化后，解析器会尝试重新定位并给出以下状态：

- `normal`：锚点稳定解析。
- `changed`：找到目标，但页面内容或结构发生变化。
- `ambiguous`：存在多个候选，需人工确认。
- `orphaned`：当前页面无法重新定位。

批注可以附加到会话编辑器。发送时，Keydex 写入不可变的网页引用快照；之后修改或删除原批注，不会静默改写已经发送的消息。

## 隐私与安全边界

- 本期不提供 Agent 点击、输入、导航、选择元素或创建批注等浏览器自动化工具。
- 网站权限默认需要用户确认；拒绝和超时均按拒绝处理。
- 恢复元数据会过滤敏感 URL 参数，不恢复 POST 请求、`file:` 等内部协议或无痕标签。
- 普通与无痕 Profile 隔离；批注截图和附件采用受控资产引用与清理策略。
- 页面桥接消息有协议版本、来源、大小和类型校验。不可注入或跨域 iframe 页面会明确降级，不伪造结构化选择结果。

## 可见性控制与发布开关

正式版默认启用浏览器和网页批注。发布层仍保留显式关闭能力，用于极端情况下快速隔离原生宿主或兼容性事故：

| 配置 | 默认值 | 用途 |
|---|---:|---|
| `VITE_KEYDEX_BROWSER_ENABLED` | `1` | 显式设为 `0` 时不注册浏览器面板 |
| `VITE_KEYDEX_BROWSER_ANNOTATIONS_ENABLED` | `1` | 显式设为 `0` 时关闭网页批注 |
| `VITE_KEYDEX_BROWSER_M0_PROBE_ENABLED` | 生产环境 `0` | 仅开发诊断使用 |

这组配置属于发布熔断，不作为普通用户设置展示。浏览器关闭时，右侧面板 Registry 会移除对应面板定义，避免只隐藏入口却残留可恢复状态。

## 故障处理

### 页面白屏

1. 确认运行的是 Tauri 桌面进程，而不是仅有 Vite 的浏览器页面。
2. 确认 Windows WebView2 Runtime 可用，并查看面板是否出现明确的宿主错误页。
3. 尝试刷新或新建普通标签；涉及 Profile 锁定时，使用新的隔离 Profile 验证，不直接删除日常数据。

### 页面周期性闪烁

周期性闪烁通常来自原生 Surface 的边界或可见性被重复同步。当前实现会缓存最后一次几何信息，并使用稳定回调，React 父组件重渲染不会在尺寸未变化时重复下发 `set_bounds` 或 `set_visibility`。

### 页面不支持结构化选择

跨域 iframe、受限页面或脚本注入失败时，文本/元素结构化选择可能不可用。此时应保留普通浏览能力并显示降级说明；不要把页面截图或坐标猜测成可稳定重定位的元素锚点。
