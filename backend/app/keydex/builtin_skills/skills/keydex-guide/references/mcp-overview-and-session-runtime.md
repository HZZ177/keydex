# MCP 能做什么与会话管理

## MCP 的用途

MCP 用于把 Keydex 连接到外部工具或服务，让 Agent 在明确范围内调用它们。一个 MCP 服务可以提供工具、登录能力，以及某些需要用户填写或批准的交互。

MCP 配置和普通 Skill 不同：Skill 主要提供工作说明，MCP 提供可实际调用的外部能力。

## 两个管理位置

- **设置中的 MCP 管理**：添加、编辑、测试、启用或删除服务，处理登录和默认策略。
- **会话中的 MCP 面板**：查看本会话可用服务和工具，并按当前任务临时调整。

全局启用表示服务可以被会话使用；会话内临时停用只影响当前会话，不等于删除全局配置。

## 推荐使用流程

1. 在设置中添加服务并填写连接信息。
2. 测试连接，确认认证有效。
3. 查看服务提供的工具，并设置展示和审批方式。
4. 进入会话，在 MCP 面板确认本次可用范围。
5. 第一次调用时仔细检查审批内容。

## 会话中没有出现

按顺序检查：

- 服务是否已启用并通过连接测试。
- 当前会话是否临时停用了该服务。
- 工具是否被设置为禁用。
- 登录或令牌是否已经失效。
- 修改设置后是否需要重新进入会话。

## 当前能力边界

服务提供的工具可以被 Agent 调用；部分服务还可提供可选提示能力。资源类能力若界面仍标记为预留或不可用，不要指导用户寻找尚未开放的入口。

MCP 不会自动获得无限权限，实际调用仍受工具策略、审批和服务自身账号权限约束。
