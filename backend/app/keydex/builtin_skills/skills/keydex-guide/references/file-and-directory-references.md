# 在会话中引用文件与目录

## 引用的作用

文件或目录引用用于明确告诉 Agent 本轮应该关注哪些项目内容。引用比只在文字中写文件名更可靠，也能减少同名文件带来的误解。

## 四种常用入口

- 在输入框输入 `@`，搜索并选择文件。
- 在 `@` 列表中进入目录继续浏览；需要引用整个目录时使用界面提示的组合键。
- 在文件树右键文件，选择“添加到聊天”。
- 在预览或批注区域，选择“添加到会话”一类入口。

目录右键的“添加该目录到会话”适合让 Agent 了解一组相关文件，但范围越大，处理时间和上下文占用通常越高。

## 发送前确认

引用成功后，输入区会显示对应文件、目录或批注标记。发送前核对：

- 是否属于当前项目。
- 是否选中了正确的同名文件。
- 是否误把整个大目录加入会话。
- 批注引用是否包含了需要的上下文。

## 普通 Chat 的限制

普通 Chat 不绑定项目，因此不能通过项目文件树或 `@` 浏览项目文件。它仍可使用普通附件，但附件和项目工作区不是同一个概念。

若任务需要反复读取、修改或定位项目文件，应切换到绑定项目的 Agent 会话或工作台。

## 文件后来发生变化

引用代表发送时选定的对象。文件被修改后，如果后续问题依赖最新内容，应重新打开或刷新文件，并在新消息中明确要求使用当前版本。批注加入会话时也应确认引用仍能对应当前文本。
