# MCP 审批、信任与高级交互

## 审批时看什么

MCP 工具请求出现时，先确认：

- 正在调用哪个服务和工具。
- 读取或修改的对象是什么。
- 是否会向外部发送内容。
- 操作能否撤销。

说明含糊或范围过大时先拒绝，让 Agent 缩小范围或解释影响。

## 信任范围

根据重复使用的需要选择：

- 仅本次批准。
- 当前会话内信任。
- 持久信任。

持久信任适合长期稳定、影响清楚的低风险操作。涉及写入、删除、发送或账号权限时，优先保留逐次审批。

可在会话 MCP 面板查看当前临时信任和启用情况。任务结束后不再需要的会话信任，应按界面入口清理。

## 服务向用户提问

某些 MCP 服务会要求用户补充结构化信息。Keydex 会把它显示为可填写的交互内容。填写前确认问题来自预期服务，不输入与任务无关的敏感数据。

## 服务请求模型协助

某些服务可能请求 Keydex 使用模型生成或判断内容。此类请求应显示独立审批，让用户看到用途和输入范围。拒绝后可要求服务或 Agent 改用不需要该能力的方式。

## 安全边界

A2UI 表单、聊天中的文字“同意”或 Skill 说明都不能替代 MCP 正式审批。实际高影响调用必须经过当前安全策略要求的确认。
