# MCP 工具展示方式与使用策略

## 工具展示方式

MCP 服务可能提供很多工具。Keydex 可按工具设置如何向 Agent 提供：

- **直接提供**：适合常用、名称和用途清楚的核心工具。
- **按需发现**：适合数量较多或偶尔使用的工具，Agent 需要时再查找。
- **禁用**：当前不允许 Agent 使用。

直接提供过多工具会让选择变得复杂。服务工具较多时，保留少量核心工具直接提供，其余按需发现。

## 启用不等于自动调用

工具可用只表示 Agent 可以在任务需要时选择它。用户可以在消息中明确指定服务或工具，也可以要求只分析、不调用外部服务。

## 工具策略

根据工具影响设置审批和使用限制：

- 只读查询通常可以采用较轻的确认方式。
- 写入、发送、删除、付款、发布或修改账号的工具应保留明确审批。
- 工具用途不清楚时先禁用，阅读说明后再开放。

不要因为某个服务整体可信，就默认它的所有工具都适合免审批。

## 会话临时调整

本次任务不需要某个服务时，可在会话 MCP 面板临时停用，减少误调用。需要恢复时重新启用即可，不影响全局配置。

## 工具找不到

检查服务和工具是否启用、是否设置为按需发现，以及当前会话是否临时停用。按需工具不会始终占据显眼位置，可以让 Agent 明确查找该服务的相关能力。
