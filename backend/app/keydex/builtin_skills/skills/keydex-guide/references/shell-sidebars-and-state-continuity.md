# 全局侧栏、面板与界面连续性

## 识别页面层级

Keydex 的常用界面可以分为三层：

- **应用层**：顶部模式入口和左侧全局导航。
- **会话层**：消息区、输入框和会话右侧栏。
- **工作台层**：项目文件树、主预览标签页和工作台助手。

用户说“右边的面板”时，先确认是会话右侧栏，还是工作台中的助手抽屉。两者入口、内容和关闭方式不同。

## 会话右侧栏

右侧栏可承载文件、预览、审阅、BTW 和 Sub-Agents 等内容。常见操作包括：

- 点击对应入口切换内容。
- 拖动边缘调整宽度。
- 使用最大化或还原入口扩大阅读区域。
- 关闭面板回到主会话。

切换标签通常不会结束当前会话。关闭右侧栏也不会删除其中关联的文件或会话记录。

## 状态保留

在不同模式间来回切换以及应用重启时，Keydex 会尽量记住用户最近选择的模式、项目、工作台文件、预览内容和面板状态。应用窗口会按当前显示器可用工作区选择初始尺寸，避开任务栏；窗口尺寸异常时先尝试系统最大化/还原，再判断是否为内容布局问题。

如果状态看起来没有恢复：

1. 确认返回的是同一项目和同一会话。
2. 查看目标文件是否仍在原路径。
3. 重新打开对应的文件、预览或侧栏标签。
4. 最后再考虑刷新页面或重启应用。

## 面板尺寸异常

面板过窄、被折叠或内容被遮挡时，先尝试拖动分隔线、退出最大化或重新打开面板。不要把布局问题误判为内容丢失。
