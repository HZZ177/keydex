# 初次使用、导航与模式选择

## 先分清当前入口

Keydex 顶部提供三个主要模式：

- **Agent**：以会话为中心，让 Agent 持续完成任务。既可以进入不绑定项目的普通 Chat，也可以进入某个项目会话。
- **工作台模式**：一边浏览和编辑项目文件，一边与助手协作，适合围绕文档或代码持续工作。
- **项目模式**：围绕项目视图组织工作。当前若标记为预览，应把它当作正在完善的产品能力。

普通 Chat 是 Agent 中“不绑定项目”的会话范围，不是第四个顶层模式。它适合通用问答、写作和不需要读取项目文件的任务。

## 根据任务选择

按以下方式推荐入口：

- 只需要交流、整理内容或使用系统级 Skill：进入普通 Chat。
- 需要 Agent 读取项目文件、运行项目内工具或使用项目级 Skill：从首页选择项目后开始会话。
- 需要频繁对照文件、预览效果、编辑源码或处理批注：进入工作台模式。
- 需要项目视图中的专门页面：进入项目模式，并留意预览提示。

## 常用导航区域

- 顶部模式入口用于在 Agent、工作台和项目模式之间切换。
- 左侧栏用于返回首页、查找历史会话和进入设置等全局页面。
- 会话右侧栏用于查看文件、预览、审阅、BTW 临时对话或 Sub-Agents。
- 工作台内部还有自己的文件区、主预览区和助手区域，不要与全局右侧栏混淆。

## 回答用户时

先说明应该进入哪个模式，再给出该模式内的具体入口。若用户只是问“怎么开始聊天”，不要要求先创建项目；若任务明确依赖项目文件，再引导选择项目。

切换模式后，Keydex 会尽量保留各模式最近的位置和面板状态。若返回后不是预期页面，先从左侧栏重新选择目标会话或项目，不要立刻让用户重置应用。
