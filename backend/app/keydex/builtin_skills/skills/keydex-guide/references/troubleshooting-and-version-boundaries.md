# 常见问题与版本边界

## 先确认四个范围

排查前先确认：

1. 当前是 Agent、工作台还是项目模式。
2. 会话是普通 Chat 还是绑定项目。
3. 当前项目和文件路径是否正确。
4. 问题发生在主会话、右侧栏、主预览还是工作台助手。

很多“内容不见了”实际是进入了另一个会话、项目或面板。

## 最小恢复顺序

按影响从小到大尝试：

1. 清除当前筛选或搜索条件。
2. 重新打开目标标签、文件或侧栏。
3. 刷新目标文件、目录、服务或选择列表。
4. 重新进入当前会话或项目。
5. 保存未提交内容后再刷新应用。
6. 最后才重启应用。

不要在有未保存源码、未提交 A2UI 或待处理审批时直接重启。

## 常见判断

- Skill 不见了：检查会话范围、目录、`SKILL.md` 格式和更高层同名覆盖；当前没有关闭系统继承或停用单个 Skill 的配置。
- `keydex.md` 没生效：确认使用准确位置和 UTF-8 普通文件、新开一轮请求，并区分普通 Chat 与项目会话；它不会出现在 Skill 选择器。
- 文件不见了：清除文件树筛选，确认当前项目，再刷新上级目录。
- 预览没更新：确认保存或冲突状态，再刷新当前文件。
- 模型不见了：检查服务、刷新模型列表并启用模型。
- MCP 不可用：测试连接、检查认证、工具策略和会话临时停用。
- 回复没有继续：查看是否在等待审批、A2UI 提交或排队输入。
- Sub-Agent 没有继续：从右侧栏打开详情，检查是否等待审批、用户输入或外部工具，不要重复委派同一任务。

## 保留证据

重复问题出现时，记录当前版本、模式、项目、会话、操作步骤和最终页面结果。错误卡片优先使用“复制错误”取得错误码、详情和任务位置；分享前仍需检查路径、业务数据、用户名和密钥。

## 版本边界

产品界面会持续变化。当前页面与本指南不一致时，以当前应用中实际可见的菜单和提示为准。项目模式若标记为预览，不应承诺其布局、入口或能力已经稳定。

本指南只回答产品使用，不用于推断源码实现。用户询问内部机制、开发接口或代码缺陷时，应检查当前仓库和当前版本。
