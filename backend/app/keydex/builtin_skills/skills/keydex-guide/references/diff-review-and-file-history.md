# Diff、文件变更与历史审阅

## 三类入口

- **会话文件变更**：Agent 或 Worker 修改文件后，从消息中的文件变更入口或右侧审阅区打开。
- **Git 本地改动**：进入 Git 面板的“本地改动”，查看工作区和暂存区差异，并执行暂存或取消暂存。
- **历史与比较**：从 Git 历史、提交详情、分支比较或修订版本文件树打开只读差异。

先确认顶部显示的仓库、文件、左右版本和“工作区/暂存区/提交”范围，避免在相似文件名或多仓库项目中审错对象。

## 统一与并排视图

审阅工具支持：

- **统一视图**：新增和删除行按单列顺序展示，适合窄窗口与快速浏览。
- **并排视图**：旧内容和新内容左右对齐，使用连接关系标出对应改动，适合逐行核对。
- **自动换行**：长行可完整阅读；关闭后更容易观察真实行结构。
- **同步滚动**：并排视图中联动左右两侧；需要独立定位时可关闭。

Git 和文件预览会记住布局、换行和同步滚动偏好；会话中的临时审阅不应假定长期保存同一选择。

## 导航与复制

多文件改动可使用“上一个/下一个文件”，单个文件内可使用“上一个/下一个差异”。未改动的大段内容会折叠，展开后仍保持原行号。

需要分享时优先“复制选中代码”；排查或提交评审需要完整机器差异时使用“复制原始补丁”。“打开文件”会跳到实际文件预览，关闭 Diff 不会删除或回退改动。

## 精确暂存

Git Diff 可以按文件、差异块或选中代码暂存和取消暂存。选中代码只处理精确文本范围，不会自动扩大到整个文件。

二进制文件、子模块、无法识别的文本编码、超大或仍被截断的差异不提供精确选择操作。先加载完整内容，或改用文件级暂存。按钮不可用时，不要把它解释为 Git 权限失败。

## 大文件与特殊内容

较大差异使用虚拟渲染，初始内容可能被截断并提供继续加载。二进制和子模块只显示摘要，不会伪造文本行差异。语法高亮用于阅读，不会改变文件内容。

## 审阅完成标准

至少确认：路径和仓库正确、改动范围符合任务、没有无关文件、删除内容可接受、文件没有未保存冲突。工具显示成功只表示某一步执行完成；最终判断以 Diff、实际文件和可见运行结果为准。
