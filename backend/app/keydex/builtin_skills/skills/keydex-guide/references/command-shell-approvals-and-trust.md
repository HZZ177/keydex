# 命令环境、审批与信任

## 选择命令环境

进入“设置 → 策略配置”，启用命令能力并选择当前电脑和项目使用的环境：PowerShell、CMD 或 Git Bash。命令写法在不同环境中可能不同，项目说明要求某一种环境时应保持一致。

Windows 用户不确定时通常使用项目当前已经工作的环境。不要把适用于 Bash 的命令直接当作 PowerShell 命令执行。

## 审批设置

当前审批策略包括“按请求”和“无条件信任”。普通使用保持按请求；无条件信任会显著减少确认，只应在用户充分理解影响、工作区和模型来源都可信时启用。

审批用于在 Agent 执行可能影响本机或外部系统的命令前让用户确认。批准时查看：

- 完整命令和工作目录。
- 会读取、修改或删除什么。
- 是否联网、安装软件或启动长期进程。
- 是否能够撤销。

命令包含不认识的路径、通配范围或高影响操作时先拒绝，让 Agent 解释或拆成更小步骤。

## 信任选择

- 一次性操作：只批准本次。
- 同一会话中重复的低风险操作：可选择会话内信任。
- 长期稳定且范围明确的规则：才考虑持久信任。

持久信任不是越多越方便。项目或工具变化后，应回到设置中审查并移除不再需要的规则。

## 命令失败

先看当前命令环境、工作目录和错误提示。不要只因为失败就扩大权限。路径或命令语法错误，应先修正；权限确实不足时，再批准最小必要范围。

停止 Agent 不会自动结束所有已经启动的外部程序。仍有进程运行时，按应用提示或明确的停止方式处理。
