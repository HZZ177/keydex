# 外部文件与只读资源

## 项目外文件

通过消息中的本机绝对路径、外部入口或其他项目链接打开文件时，Keydex 会按该文件自己的真实位置读取，不再把路径错误拼接到当前项目。它会作为本地文件标签或预览打开，不会自动插入当前项目文件树。

能否读取和编辑取决于“设置 → 策略配置 → 访问范围”和磁盘权限。“工作区内只读”或“工作区内信任”不会自动获得任意项目外路径；“完全访问”下，允许范围内的外部文本文件可进入源码编辑和文件变更历史。不要把“项目外”一律描述为只读，也不要因为预览成功就推断 Agent 已获得写入权限。

Windows 绝对路径、含空格路径和经过 URI 编码的本机文件链接都可以从会话中打开。链接打不开时先复制并核对完整路径、盘符、文件是否存在和访问策略；不要先切换项目破坏当前工作上下文。

## Skill 资源

Skill 附带的参考资料和资源可以从会话或 Skill 入口打开。内置 Skill 资源由应用管理，应按只读内容使用；不要指导用户直接修改安装内容。

系统级或项目级 Skill 的源文件属于对应用户或项目，但通过只读资源预览打开时，当前预览仍可能不提供编辑。要修改时，应从实际文件位置打开，并确认改动范围。

## 只读状态下的限制

只读资源通常不能：

- 在源码模式中保存修改。
- 创建依赖可写文档的批注。
- 使用会覆盖原文件的操作。

仍可进行阅读、搜索、复制，并在允许时添加为会话参考。

## 外部文件变化

外部程序修改文件后，可使用当前文件的重新加载入口读取最新磁盘内容；移动或删除后，预览可能暂时保留最后看到的内容。文件不存在时，应明确区分“预览里还有旧内容”和“磁盘文件仍存在”。重新加载会放弃当前预览中的未保存编辑时，应先确认冲突提示。

## 安全提示

把本机绝对路径、外部文档或 Skill 资源发送到会话前，检查是否包含账号、密钥或个人目录信息。只引用当前任务需要的内容。
