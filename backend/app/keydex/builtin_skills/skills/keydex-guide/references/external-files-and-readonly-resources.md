# 外部文件与只读资源

## 项目外文件

通过外部入口打开、不属于当前项目的文件，可以在预览区查看，但不会自动出现在项目文件树中。它也不应被当作当前项目的普通可编辑文件。

需要长期编辑或引用时，建议先确认是否应该把文件放入项目，或切换到包含该文件的正确项目。

## Skill 资源

Skill 附带的参考资料和资源可以从会话或 Skill 入口打开。内置 Skill 资源由应用管理，应按只读内容使用；不要指导用户直接修改安装内容。

系统级或项目级 Skill 的源文件属于对应用户或项目，但通过只读资源预览打开时，当前预览仍可能不提供编辑。要修改时，应从实际文件位置打开，并确认改动范围。

## 只读状态下的限制

只读资源通常不能：

- 在源码模式中保存修改。
- 创建依赖可写文档的批注。
- 使用会覆盖原文件的操作。

仍可进行阅读、搜索、复制，并在允许时添加为会话参考。

## 外部文件变化

外部程序修改、移动或删除文件后，预览可能暂时保留最后看到的内容。使用刷新确认当前磁盘状态。文件不存在时，应明确区分“预览里还有旧内容”和“磁盘文件仍存在”。

## 安全提示

把本机绝对路径、外部文档或 Skill 资源发送到会话前，检查是否包含账号、密钥或个人目录信息。只引用当前任务需要的内容。
