# `.keydex` 的内置、系统级与项目级能力

## 三个范围

- **内置层**：随 Keydex 发布，由应用维护。目前提供只读内置 Skill。
- **系统级层**：保存在当前用户目录，适合多个项目共同使用的工作约定和 Skill。
- **项目级层**：保存在项目中，只服务于该项目，并可随项目一起管理。

默认位置：

- 系统级 Skill：`%USERPROFILE%\.keydex\skills`
- 系统级工作区说明：`%USERPROFILE%\.keydex\keydex.md`
- 项目级 Skill：`<项目目录>\.keydex\skills`
- 项目级工作区说明：`<项目目录>\.keydex\keydex.md`

每个 Skill 使用自己的子目录。

## 同名优先级

同名时，最终生效顺序是：

1. 项目级
2. 系统级
3. 内置

选择面板采用覆盖式显示，只展示胜出的那一项。想恢复低一级版本时，应改名或移走高优先级同名 Skill，而不是在列表里寻找第二份。高层同名 Skill 即使格式无效，也会阻止系统静默降级到低层同名版本，以免实际执行与项目意图不一致。

## 固定继承与配置边界

项目会话固定加载内置、系统级和项目级中当前能力支持的范围，不提供关闭系统级继承的目录开关。Skill 的优先级固定为项目级高于系统级、系统级高于内置。

`.keydex` 下不使用 JSON 文件控制能力、版本、启停或继承；即使保留旧的 `keydex.json`，Keydex 也会完全忽略它。能力清单和格式版本随应用代码发布，文件内容变化由运行时自动监听。

## `keydex.md` 的分层方式

`keydex.md` 用于提供长期有效的工作区背景和约定，不使用同名覆盖：

1. 普通 Chat 只加载系统级 `keydex.md`。
2. 项目会话先加载系统级，再加载项目级 `keydex.md`。
3. 缺失或空白文件会被跳过；无有效文档时不会产生空注入。

文档会在每次模型请求时作为临时工作区上下文加入，不写入消息历史。它的优先级低于 Keydex 平台指令、安全规则、已激活 Skill 和用户当前请求，不能通过文档内容扩大工具或文件权限。

## 普通 Chat

普通 Chat 没有项目层，因此 Skill 优先级是系统级高于内置，并可读取系统级 `keydex.md`。它不需要为了使用系统级能力强制创建项目，也不会因此获得项目文件工具。

## 所有权

不要直接修改内置 Skill：应用安装或更新会用发布版本覆盖它。需要定制时，在系统级或项目级创建同名 Skill，让更高优先级版本覆盖内置内容。
