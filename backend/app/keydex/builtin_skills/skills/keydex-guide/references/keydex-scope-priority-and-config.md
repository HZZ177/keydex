# 内置、系统级、项目级 Skill

## 三个范围

- **内置 Skill**：随 Keydex 发布，由应用维护，所有用户都可以获得基础能力。
- **系统级 Skill**：保存在当前用户目录，适合多个项目共同使用的个人或团队工作方式。
- **项目级 Skill**：保存在项目中，只服务于该项目，并可随项目一起管理。

默认位置：

- 系统级：`%USERPROFILE%\.keydex\skills`
- 项目级：`<项目目录>\.keydex\skills`

每个 Skill 使用自己的子目录。

## 同名优先级

同名时，最终生效顺序是：

1. 项目级
2. 系统级
3. 内置

选择面板采用覆盖式显示，只展示胜出的那一项。想恢复低一级版本时，应改名、移走或禁用高优先级同名 Skill，而不是在列表里寻找第二份。

## 项目是否继承系统级 Skill

项目默认可以使用系统级 Skill。项目需要完全独立时，可在项目 `.keydex` 配置中关闭系统级 Skill 继承。关闭后，项目会话仍可使用项目级和内置 Skill。

项目也可以通过自身配置启用或停用特定 Skill。修改后若列表没有变化，重新进入项目会话或使用当前提供的刷新入口。

## 普通 Chat

普通 Chat 没有项目层，因此优先级是系统级高于内置。它可以使用系统级 Skill，不需要为了使用 Skill 强制创建项目。

## 所有权

不要直接修改内置 Skill：应用安装或更新会用发布版本覆盖它。需要定制时，在系统级或项目级创建同名 Skill，让更高优先级版本覆盖内置内容。
