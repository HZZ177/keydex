# 设置页面地图与保存行为

## 入口层级

从全局设置入口进入后，当前页面按以下分组组织：

| 页面 | 主要内容 |
|---|---|
| 常规 | 关闭行为、回复进行中再次发送的默认方式 |
| 外观 | UI 字体；浅色/深色主题入口位于设置侧栏底部 |
| 供应商配置 | 模型服务地址、凭据与服务实体 |
| 模型配置 | 已启用模型、默认模型和快速模型 |
| 扩展功能 | 标题、重复保护、上下文整理、A2UI、Web 等增强能力 |
| MCP服务器 | MCP 服务、传输、认证、工具和策略 |
| 策略配置 | 命令环境、审批策略、文件访问、编辑方式和信任规则 |
| 用量统计 | 用量趋势、明细与导出 |
| 项目管理 | 已知项目及项目相关管理 |
| 归档管理 | 已归档会话的查看与恢复 |
| 关于 | 当前版本、更新日志、检查和安装更新 |

描述入口时使用“设置 → 页面名 → 控件名”的完整层级。旧版本中的“模型服务”“安全设置”等泛称，应换成当前可见页面名。

## 哪些设置实时保存

普通开关、下拉选择和输入项通常在修改后自动保存，不需要再寻找全局“保存”按钮。保存期间界面会先反映新值；若校验或写入失败，会提示错误并回到最近一次已确认的值。快速连续修改时以最后一次有效选择为准。

输入仍处于无效状态时不会写入。例如必填项为空、数值越界或格式不正确，应先按页面提示修正。

## 哪些操作需要明确确认

供应商和 MCP 服务器等完整实体配置仍通过表单的保存/确认动作提交。删除、覆盖、导入以及会扩大权限的动作也会单独确认，不属于普通实时保存。

远程连接验证是独立动作，不是保存副作用：MCP 和 Web 使用页面中的“测试连接”；模型供应商先“刷新模型”，再对具体模型使用“测试”。保存配置不等于已经连通，也不应把连接失败解释为保存失败。

## 修改后如何确认

- 普通偏好：切换到其他页面再返回，确认值仍在。
- 模型或供应商：刷新模型列表，并确认至少启用一个可用模型。
- MCP：测试连接，再到会话中确认工具状态。
- 策略：用最小读写或命令操作验证，不要直接扩大到全盘权限。
- 更新：到“关于”查看当前版本和更新状态。
