# Skill 选择、显示与激活

## Skill 的作用

Skill 是一组针对特定任务的说明和资源，用来告诉 Agent 应该采用什么工作方式。它适合可重复的专业流程、产品手册或固定输出规范。

## 从哪里选择

可以从首页、会话输入区的 `/skill`，或当前提供的 Skill 选择入口查看可用项。选中后，Skill 会作为本次会话或任务的指导来源。

普通 Chat 可以使用最终生效的系统级和内置 Skill，但没有项目级 Skill。绑定项目的会话还可以使用项目中的 Skill。

## 列表为什么只有一份同名 Skill

同名 Skill 可能同时存在于项目、系统和应用内置范围。选择面板只显示最终生效的一项，不会把三份重复内容都列出来：

- 项目级优先于系统级。
- 系统级优先于内置。

因此，用户看到的是当前会话真正会使用的版本。可根据界面显示的来源判断它来自项目、系统还是内置。

## Skill 没有自动使用

Skill 的说明与当前问题匹配时，Agent可以自动采用；用户也可以显式选择。未激活时按顺序检查：

1. Skill 是否出现在当前会话的选择列表中。
2. 当前会话是否属于预期项目。
3. Skill 目录是否包含合法的 `SKILL.md`、名称和描述。
4. 同名 Skill 是否被更高优先级版本覆盖或阻挡。
5. Skill 内容是否清楚说明了适用问题。

`.keydex/keydex.md` 与 Skill 的触发方式不同：它不出现在 Skill 选择列表，也不需要调用 `load_skill`。存在有效内容时，Keydex 会按当前会话范围自动把它作为临时工作区上下文提供给模型。

## 使用后确认

需要知道最终采用了哪一份时，查看 Skill 名称和来源提示。不要只根据磁盘上存在某个文件夹，就判断它一定已经被当前会话使用。
