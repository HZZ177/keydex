# 联网搜索与回答来源

## 开启联网能力

在扩展或联网相关设置中启用 Web 能力，并选择当前支持的搜索服务。需要凭据时，通过设置保存，不要把密钥直接发在聊天消息中。

配置完成后使用测试入口确认连接可用。测试失败时先检查服务、网络和凭据，不要连续在会话中重试同一搜索任务。

## 搜索与抓取

- **搜索**用于发现相关网页和候选来源。
- **抓取/打开**用于读取具体页面内容。

用户要“最新”“现在”“查一下”或需要精确来源时，应使用联网能力；只基于项目文件的问题不必默认联网。

## 回答来源

启用来源展示后，回答中应给出可核对的网页链接。判断可靠性时优先：

- 官方文档和原始发布方。
- 与问题直接相关的页面。
- 明确的发布时间或更新时间。
- 多个来源之间能够相互印证的信息。

搜索结果摘要不等于完整页面结论。需要精确条款、价格、版本或日期时，打开原页面确认。

## 常见限制

登录墙、付费墙、验证码或站点限制可能导致内容无法读取。此时应说明限制，并让用户提供页面内容或改用可访问的官方来源。

联网答案可能随时间变化。涉及高影响决策时，应注明查询时间并让用户核对最终来源。
