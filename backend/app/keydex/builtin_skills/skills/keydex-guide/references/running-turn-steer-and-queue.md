# 回复进行中时的引导与排队

## 两种发送方式

当 Agent 正在回复时，新输入可以有两种用途：

- **引导当前回复**：把补充要求交给正在进行的任务，例如“只改这个页面”或“先别运行完整测试”。
- **加入队列**：等当前回复结束后，再作为下一轮消息发送。

通用设置中可以决定回复进行时的默认发送方式。按住 Ctrl 再发送，通常会临时使用另一种方式；应以输入区当前提示为准。

## 什么时候使用引导

适合立即纠正当前方向：

- 补充遗漏的限制。
- 缩小正在执行的范围。
- 指出刚发现的事实或文件。
- 要求先停在分析，不继续修改。

引导并不保证撤销已经完成的操作。涉及文件改动或外部操作时，应查看当前进度和最终结果。

## 什么时候加入队列

适合与当前任务相对独立的后续事项：

- 当前修复完成后再写总结。
- 完成本轮后再处理另一个文件。
- 先验收，再执行下一项任务。

队列中的消息可以根据界面提供的入口调整顺序、编辑、改变发送方式或删除。修改队列时，确认没有误删仍需要执行的要求。

## 手动停止后的处理

停止当前回复后，先查看已完成到哪一步，再选择继续、补充说明或重新发起。不要默认停止会自动回滚文件改动。

如果停止时存在排队消息，检查队列是否仍保留，再决定是否发送。重要任务应在恢复消息中明确指出“保留已经完成的工作，从某一步继续”。
