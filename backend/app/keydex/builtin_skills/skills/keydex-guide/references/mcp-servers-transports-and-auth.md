# MCP 服务、连接方式与登录认证

## 添加服务

进入设置中的 MCP 页面，新增服务并选择与服务提供方一致的连接方式。常见方式包括：

- **本地命令（stdio）**：由本机命令启动服务。
- **HTTP**：连接到网络服务地址。
- **SSE**：连接到使用持续事件流的服务地址。

不要凭名称猜连接方式，应以服务提供方的安装说明为准。

## 本地命令需要填写什么

通常需要命令、参数、环境变量和可选工作目录。保存前确认：

- 命令在当前电脑上可以使用。
- 参数顺序与服务说明一致。
- 工作目录确实存在。
- 密钥等敏感值不要写进可分享的普通文本。

## 网络服务需要填写什么

填写完整服务地址，并按服务要求选择认证方式。常见认证包括：

- 无认证。
- 自定义请求头。
- 通过环境变量提供 Bearer 凭据。
- OAuth 登录。

使用环境变量能避免把密钥直接写入导出配置。OAuth 应通过正式登录入口完成，不要把账号密码交给 Agent 输入。

## 测试和启用

保存前后都可使用连接测试。测试成功表示当前连接与认证基本可用，不代表每个工具操作都会被账号授权。

编辑连接方式或认证后重新测试。暂时不用的服务可停用；确认不再需要时再删除。

## 登录过期

OAuth 或其他凭据失效时，使用重新登录或刷新入口。需要彻底断开账号时，执行撤销或退出授权，而不是只停用服务。
