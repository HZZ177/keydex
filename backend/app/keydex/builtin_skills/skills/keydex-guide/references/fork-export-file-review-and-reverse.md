# 分叉、导出、文件审阅与回退操作

## 分叉会话

当用户希望从某个历史节点尝试另一种方案，同时保留原讨论时，使用分叉。分叉会创建独立会话，原会话不会被改写。

分叉后先确认新会话的项目、目标和必要上下文是否正确。后续消息不会自动同步回原会话。

## 导出会话

需要备份、分享或离线阅读时，将会话导出为 Markdown。导出前注意：

- 检查是否包含路径、命令、账号信息或其他敏感内容。
- 图片、附件和交互内容可能以文字说明或引用形式呈现，分享前先打开导出文件确认。
- 导出不会删除或归档原会话。

## 文件审阅

Agent 或 Worker 修改文件后，可从消息中的文件变更入口或右侧审阅区查看改动。主 Agent 会话统一记录子会话产生的文件变更，不需要逐个子会话导出补丁。审阅时重点确认：

- 文件路径是否正确。
- 新增和删除内容是否符合要求。
- 是否包含无关文件。
- 是否还有未保存或冲突内容。

看到工具执行完成，不等于所有改动已经满足需求；以审阅内容和实际页面结果为准。

审阅区支持统一和并排布局、自动换行、差异导航、文件导航、复制选中代码和复制原始补丁。需要暂存 Git 改动时进入 Git 面板；会话审阅本身不等于 Git 暂存。

## 回退或撤销

只有界面明确提供“撤销”“回退”或同等入口时，才指导用户使用。执行前说明它影响哪些文件或操作，并先保存仍需保留的内容。

如果没有可见回退入口，不要声称系统会自动恢复。可以让 Agent 基于审阅结果做反向修改，但应明确目标文件和要恢复的内容。

停止生成、拒绝审批或删除消息都不等于撤销已经写入文件的修改。
