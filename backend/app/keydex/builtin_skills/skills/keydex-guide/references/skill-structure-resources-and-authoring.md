# Skill 内容、资源与创建方式

## 最小组成

一个 Skill 目录至少包含 `SKILL.md`。这个文件说明：

- Skill 的名称和适用场景。
- Agent 应遵循的工作方式。
- 需要时应读取哪些附加资料。

说明应围绕用户任务写清触发场景和预期结果。不要只写产品名或一句过于宽泛的描述。

## 附加资源

内容较多时，可以在 Skill 目录中加入参考资料、模板或任务需要的其他资源。主 `SKILL.md` 负责导航，让 Agent 只在需要时读取相关资料。

建议按一个明确功能或任务拆分资料，例如“会话管理”“文件批注”“模型设置”，而不是放一个只有文件名列表的薄索引。

## 放到哪个范围

- 多个项目都要用：放到系统级目录。
- 只服务一个项目：放到项目的 `.keydex\skills`。
- 应用发布方统一提供：作为内置 Skill 随应用发布。

用户自己创建内容时，优先系统级或项目级，不要编辑安装目录中的内置版本。

## 编写产品指导 Skill

面向产品用户的 Skill 应优先写：

1. 从哪里进入。
2. 怎样操作。
3. 操作后会看到什么。
4. 有哪些前置条件和限制。
5. 出错后怎样保留工作并恢复。

不要把内部接口、源码模块或实现状态当作普通用户手册内容。

## 修改后验证

确认 Skill 能在正确范围的选择列表中出现，名称和描述清楚，同名覆盖符合预期，并实际提一个匹配问题验证 Agent 能找到正确资料。
