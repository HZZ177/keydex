# A2UI 使用流程、继续与调试入口

## A2UI 是什么

A2UI 是 Agent 在消息中提供的可操作内容。当前常见形式包括图表、选择、表单和可编辑表格。它让用户直接在会话中查看信息或提交结构化答案。

## 一次完整交互

1. Agent 发出交互内容。
2. 用户查看说明，完成选择、填写或编辑。
3. 点击提交，或在不继续时选择取消。
4. Agent 根据本次结果继续任务。

有些任务会在 A2UI 处等待用户，不提交就不会自动继续。提交后应等待 Agent 的确认和后续结果，不要重复点击提交。

## 修改与纠正

如果交互内容允许纠正，按当前界面提示修改后重新提交。先确认是修改尚未提交的内容，还是对已经提交的结果发起纠正；两者对后续任务的影响不同。

界面显示为只读或已完成时，可以在普通消息中说明需要调整什么，让 Agent 重新发起合适的交互。

## 取消与恢复

取消表示本次不按当前交互继续，并不一定结束整个会话。之后可通过普通消息说明新的要求，或让 Agent 重新提供交互内容。

会话重新打开后，如果仍显示等待交互，先检查是否已有提交结果；不要在状态不明确时连续提交同一内容。

## 调试入口

设置中可提供 A2UI 调试入口，用于查看交互内容和提交结果。它主要用于排查展示或交互问题，普通使用无需开启。

用户反馈 A2UI 异常时，优先收集可见类型、填写步骤、提交后的页面结果和错误提示，不必要求用户理解内部数据格式。

## 安全边界

A2UI 不能替代命令执行、文件访问或 MCP 操作的安全审批。即使用户在表单中选择“允许”，需要审批的实际操作仍应出现正式审批入口。
