# MCP 导入导出、审计与故障排查

## 导入配置

导入前先查看服务名称、连接地址或命令、工具范围和认证方式。来自他人的配置可能引用本机不存在的命令、目录或环境变量，导入后应逐个测试。

导入不会替用户完成 OAuth 登录，也不应依赖配置文件携带明文密钥。

## 导出配置

导出适合迁移服务结构和非敏感设置。分享前打开导出内容检查：

- 不包含令牌、密钥或账号密码。
- 本机绝对路径是否需要改成说明或占位符。
- 是否只包含打算分享的服务。

接收方仍需配置自己的环境变量和登录授权。

## 查看审计信息

审计入口用于确认 MCP 服务何时被测试、调用、批准、拒绝或修改。排查问题时记录时间、会话、服务、工具和可见结果，避免只说“刚才 MCP 不工作”。

## 常见故障顺序

1. **无法连接**：检查连接方式、命令或地址，再运行测试。
2. **认证失败**：重新登录、刷新凭据或确认环境变量。
3. **服务可用但工具不见了**：检查工具是否禁用或按需发现。
4. **会话里不能用**：检查会话是否临时停用了服务。
5. **调用被拒绝**：查看审批、信任和外部账号权限。
6. **改了设置仍无变化**：重新进入会话后再确认。

## 删除与撤销

删除 Keydex 中的服务配置不一定会撤销外部账号授权。若使用 OAuth 或服务端令牌，应同时使用撤销授权入口或到服务方账号中移除授权。
