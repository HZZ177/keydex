# 文件访问范围与编辑方式

## 文件访问范围

文件访问设置决定 Agent 可以读取或修改哪些位置。绑定项目的会话通常以当前项目为主要工作范围；访问项目外文件时，可能需要额外批准。

设置时遵循最小范围：

- 只处理当前项目，就不要开放整个磁盘。
- 只需读取时，不要授予修改权限。
- 含密钥、个人资料或系统文件的目录应保持受限。

## 临时访问与长期规则

一次性外部文件优先使用单次批准。只有反复使用且范围清楚的目录才考虑长期允许。项目结束或目录迁移后，清理不再需要的规则。

## 文件编辑方式

设置中可能提供不同的文件编辑兼容方式，用于适配模型或服务对文件修改工具的支持。普通用户保持推荐默认值即可；只有当前模型服务明确要求，或编辑持续失败并有对应说明时才切换。

切换编辑方式不会自动修复已有内容冲突，也不会撤销之前的修改。调整后用一个小文件或小范围改动验证。

## 访问被拒绝

1. 确认目标文件是否属于当前项目。
2. 检查文件是否只读、已移动或被删除。
3. 查看是否有待处理的访问审批。
4. 只为所需文件或目录增加权限。

不要为了绕过单个错误直接开放整个用户目录或磁盘。

## 修改后的确认

Agent 表示写入成功后，仍应在预览或审阅区查看最终内容。文件访问获批只表示允许操作，不表示任务已经正确完成。
