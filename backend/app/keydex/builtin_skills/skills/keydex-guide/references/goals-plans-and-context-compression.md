# 目标、计划与上下文整理

## 会话目标

目标用于让 Agent 在多轮任务中持续记住要达成的结果。一个会话同一时间只保留一个活动目标。

可以根据界面提供的入口：

- 创建目标。
- 查看目标内容和当前状态。
- 暂停或恢复。
- 编辑目标。
- 删除不再需要的目标。

暂停目标不会删除会话内容；恢复后应先查看此前进展。删除目标只移除目标约束，不等于撤销已经完成的工作。

## 什么时候设置目标

适合持续多轮、需要验收或容易偏离范围的工作，例如完整重构、逐项迁移、持续排查。一次性简单问答不必强制创建目标。

目标应描述最终结果和边界，不要把每个临时操作都写进目标。具体步骤可以交给本轮计划。

## 计划

计划用于展示较复杂任务的步骤和进度。用户可以根据计划判断当前在做什么、还剩什么，以及是否需要中途调整范围。

计划不是完成证明。验收时仍应查看最终内容、测试或页面结果。

## 上下文整理

会话很长时，可以使用 `/compact` 或界面提供的上下文整理入口。它用于保留关键结论并压缩较早内容，让后续对话继续进行。

执行前建议：

- 确认重要限制、未完成项和文件位置已经明确。
- 对极其关键的原文先单独保存或导出。
- 整理后检查目标、当前任务和下一步是否仍准确。

上下文整理不是删除会话，也不等于归档。
