# 工作台布局与预览标签页

## 三个主要区域

工作台围绕项目文件组织：

- 左侧是项目文件树，用于筛选、展开、定位和打开文件。
- 中间是主预览区，用标签页承载已打开的文件。
- 助手以胶囊、输入区、展开面板或抽屉形式出现，用于围绕当前项目协作。

区域之间的分隔线可以调整。内容过窄时先拖动分隔线，不要误认为文件没有加载完整。

## 打开和切换文件

在文件树中选择支持的文件后，会在主预览区打开。会话中的绝对本机路径和其他项目的文件也可以打开为本地文件标签，但不会加入当前项目树。多个文件以标签页保留，点击标签可快速切换。

标签页常见操作包括：

- 关闭当前标签。
- 关闭左侧标签。
- 关闭右侧标签。
- 关闭其他标签。
- 关闭全部标签。

关闭标签只关闭当前预览，不会删除磁盘文件。文件有未解决的编辑冲突或其他提示时，应先处理提示再批量关闭。

## 主预览与右侧预览

工作台主预览用于持续查看和编辑项目文件。Agent 会话中的右侧预览更适合跟随消息临时查看。用户说“把文件打开到右边”或“在工作台中打开”时，要区分目标区域。

## 恢复布局

从其他模式或重启后返回工作台时，Keydex 会尽量保留最近项目、标签页、当前文件和面板位置。项目、当前文件与启动时用户选择会被分别恢复；若没有恢复：

1. 确认当前工作台仍是同一项目。
2. 在文件树重新打开目标文件。
3. 检查助手或预览区是否被折叠。
4. 最后再刷新应用。

不要用刷新作为第一步，因为刷新可能打断正在编辑或查看的位置。
