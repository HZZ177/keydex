# 预览格式、源码与分屏模式

## 支持的常见内容

工作台或右侧预览可根据文件类型显示不同内容，包括：

- Markdown、HTML 与纯文本。
- JSON 和常见代码文件。
- Diff 差异内容。
- Mermaid 图表。
- 图片。

不支持的格式可能只提供基础文本查看、外部打开，或提示无法预览。

## Markdown 与 HTML

对适合渲染的文件，可在以下模式间切换：

- **预览**：查看排版后的内容。
- **源码**：查看并编辑原始文本。
- **分屏**：同时查看源码与预览，适合边写边核对效果。

不同文件或只读资源可能只开放其中一部分模式。没有源码入口时，先确认文件是否只读、是否属于当前项目，以及文件访问策略是否允许写入。

Markdown 在预览、源码和分屏间切换时会尽量保留当前阅读位置。切回后若定位偏移，先等待内容完成渲染，再从大纲或搜索结果跳转，不要用刷新作为第一步。

## 本地 HTML 与 Vite 页面

在项目文件树中打开 `.html` 文件时，预览会从文件所在工作区提供受控的本地页面地址，使相对路径引用的脚本、样式和图片可以按项目结构加载。页面脚本可在预览 iframe 中运行，但仍受预览隔离和工作区路径边界限制。

消息中的 HTML 代码块可以展开或全屏预览；内容若引用本机 loopback Vite 开发地址，先确保对应开发服务器正在运行。Keydex 不会替用户自动启动 Vite，也不会把任意项目外路径当作工作区静态资源。

HTML 空白或样式缺失时依次检查：源码是否保存、相对资源路径是否以 HTML 文件位置为基准、资源是否仍在当前工作区、Vite 服务是否可访问，以及页面是否依赖被浏览器安全策略阻止的远程内容。需要排查脚本时切到源码，不要把渲染失败等同于源文件为空。

## 内容搜索与复制

需要查找正文时，使用预览区的搜索入口，而不是文件树名称筛选。搜索结果可逐项跳转。

复制时注意当前所在模式：预览模式复制的是渲染后的可见内容，源码模式复制的是原始文本。需要保留 Markdown 标记时使用源码模式。

## 关闭预览

关闭标签或预览面板不会删除文件。再次从文件树或消息入口打开即可。

## 显示异常

如果预览内容与源码不一致：

1. 检查源码是否仍有未保存或冲突提示。
2. 切换到源码确认真实文本。
3. 刷新该文件。
4. 对 HTML 等内容，确认本地资源路径、开发服务器和浏览器安全限制。

不要仅凭渲染效果判断文件已经丢失或被改写。
