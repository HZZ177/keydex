# 通用设置、外观与应用更新

## 通用设置

通用设置中可调整应用关闭行为，以及 Agent 回复进行中再次发送消息时的默认方式。

关闭行为应根据使用习惯选择：真正退出应用可能停止后台任务；最小化或保留后台运行则可能继续占用本机资源。修改前查看页面说明。

回复进行时的默认发送方式可设为引导当前回复或加入队列。输入区提示会显示当前行为，按住 Ctrl 通常可临时反转。

## 外观

进入“设置 → 外观 → UI 字体”选择系统默认、Maple Mono 或 JetBrains Mono。首次选择非系统字体时会下载所需资源并显示进度；字体变更只影响应用显示，不会修改文件内容。

设置侧栏底部的“浅色/深色”按钮用于切换主题，不在“外观”页面的字体卡片中。主题会影响应用界面和预览配色，不会改写文档或代码。

出现文字截断或排版异常时：

1. 切回“系统默认”字体。
2. 调整面板宽度。
3. 若下载字体失败，检查网络后重新选择；已缓存字体可直接启用。

## 关于与更新

进入“设置 → 关于”查看当前应用版本。这里提供“更新日志”“检查更新”，发现新版本后提供“查看并更新”。更新日志可浏览当前版本和较早版本；离线时可能继续展示本地缓存的发布说明，并可在需要时跳到 GitHub 查看原文。

手动“检查更新”会打开更新结果或更新对话框。应用启动时发现新版本也可能主动提示；运行期间的周期检查保持静默，只更新全局设置入口和“关于”页面的更新徽标，不应突然打断当前操作。后台检查失败不会清除已经可见的新版本状态。

下载或安装失败时保留对话框中的错误和重试入口，确认代理、网络和磁盘空间后重试。更新重启前完成或停止重要任务，并保存正在编辑的文件；重启不会替代对未保存文件的确认。

应用更新会更新内置 Skill 等发布方管理内容，但不应覆盖用户的系统级 Skill 或项目级 Skill。若存在同名用户定制，更新后的内置版本仍会被高优先级定制覆盖。

## 更新后检查

- 查看重要项目和上次工作台文件是否仍按原位置打开。
- 检查自定义系统级或项目级 Skill 的生效来源。
- 确认系统级与项目级 `keydex.md` 仍在用户目录和项目目录中；它们不应被应用更新覆盖。
- 若版本说明涉及模型、预览或策略，再针对对应能力做最小验证，不需要每次升级后重配全部服务。

不要通过直接修改安装目录来保留定制内容，应把定制放在用户或项目范围。
