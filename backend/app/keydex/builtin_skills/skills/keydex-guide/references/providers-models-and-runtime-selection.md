# 模型服务、默认模型与会话切换

## 配置模型服务

进入“设置 → 供应商配置”，添加与服务兼容的连接信息。常见配置包括服务名称、接口地址和密钥。实体配置需要明确保存；保存后使用“刷新模型”验证地址和认证，再对具体模型使用“测试”检查可用性。密钥保存后会以遮罩形式显示，不应要求用户在会话中重复粘贴完整密钥。

保存后刷新模型列表，并只启用实际需要使用的模型。服务连接成功但没有启用任何模型时，首页和会话仍可能没有可选项。

## 默认模型与快速模型

进入“设置 → 模型配置”，可以选择：

- **默认模型**：新会话和普通任务优先使用。
- **快速模型**：适合标题生成、轻量整理或产品明确使用快速模型的辅助任务。

两者可以相同，也可以按成本、速度和质量分开。默认项失效时，应重新选择当前已启用的模型，不要静默换成未知模型。

## 会话中切换

会话输入区或运行设置中的模型选择器用于调整当前任务的模型。切换通常影响之后的请求，不会改写已经生成的消息。

任务正在回复时不要频繁切换；先等待结束或停止，再确认下一轮使用的模型。

## 模型列表为空

1. 检查模型服务是否启用。
2. 检查地址和密钥是否有效。
3. 刷新该服务的模型列表。
4. 确认至少启用一个模型。
5. 重新设置默认模型并返回会话。

## 安全与成本

不要在截图、导出文件或会话消息中暴露密钥。使用第三方模型服务时，用户输入和附件可能会发送到对应服务，应遵循该服务的数据与计费规则。
