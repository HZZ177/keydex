# 源码编辑、自动保存与冲突处理

## 进入源码编辑

对支持编辑的项目文件，切换到“源码”或“分屏”模式后直接修改。允许范围内的本机外部文本文件也可编辑；只读 Skill 资源和文件访问策略未覆盖的内容不会开放编辑。

## 保存方式

编辑停止一小段时间后，Keydex 会自动保存；离开编辑区或使用界面支持的保存快捷键，也会触发保存。完成重要修改后，应观察保存状态再关闭标签或应用。

自动保存不等于每个按键都立即写入磁盘。刚输入完就被外部工具读取时，先等待保存提示完成。

## 外部修改冲突

如果文件在编辑期间被其他程序或 Agent 修改，Keydex 会提示内容冲突。根据实际意图选择：

- **保留当前草稿**：先保住正在编辑的内容，之后再比较处理。
- **加载磁盘内容**：放弃当前草稿，使用外部最新版本。
- **用当前内容覆盖**：确认草稿应成为最终版本时覆盖磁盘内容。

覆盖可能丢失外部修改。选择前先复制重要片段，或在其他位置对比两份内容。

## 文件被删除或重建

预览中的最后内容可能暂时保留，方便查看和恢复，但这不代表磁盘文件仍然存在。相同路径重新创建后，可刷新并继续使用。

文件被删除时不要继续假设自动保存会重新创建它；先确认是要恢复文件还是关闭旧预览。

外部程序已写入新内容但当前预览仍旧时，使用文件的重新加载入口。重新加载前先确认是否有未保存草稿；它用于读取最新磁盘内容，不是撤销历史。

## 处理保存失败

1. 检查文件是否只读、已删除或不在允许的编辑范围。
2. 查看是否存在冲突提示。
3. 复制未保存内容作为临时备份。
4. 只刷新目标文件，重新打开后再尝试。

在保存状态不明确时，不要直接关闭所有标签。
