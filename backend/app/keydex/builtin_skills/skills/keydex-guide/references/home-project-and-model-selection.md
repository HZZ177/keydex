# 首页、项目与模型选择

## 从首页开始

首页用于决定本次任务是否绑定项目：

- 选择普通 Chat，可直接开始不依赖项目文件的会话。
- 选择已有项目，可让会话使用该项目的文件、设置和项目级 Skill。
- 项目不在列表中时，使用添加项目入口，通过文件夹选择器或路径添加。

添加项目只是在 Keydex 中登记这个工作位置，不等于上传整个项目。后续可在项目管理中查看或移除登记项。

## 开始前的可选项

根据首页当前提供的控件，可以在发送第一条消息前选择：

- 本次会话使用的模型。
- 希望明确启用的 Skill。
- 需要持续跟踪的目标。

这些选项不是每次都必须设置。用户只是想快速开始时，使用已有默认值即可。

## 项目选择建议

- 一个任务只涉及一个项目时，直接进入对应项目。
- 需要引用多个位置的文件时，先确认当前项目是否覆盖主要工作目录；不要默认当前会话能访问任意磁盘位置。
- 项目改名、移动或删除后，如果首页入口失效，可在项目管理中移除旧记录并重新添加新位置。

## 模型不可选时

按顺序检查：

1. “设置 → 供应商配置”中是否已经添加并保存可用服务。
2. 该服务下是否有已启用的模型。
3. “设置 → 模型配置”中的默认模型是否仍然存在。
4. 回到首页或新建会话后重新打开模型选择器。

不要把“列表中没有模型”直接判断为网络故障；它也可能是服务未配置、模型未启用或默认项已失效。
