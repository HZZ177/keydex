# Skill 安全、诊断与更新规则

## 使用前检查来源

Skill 会影响 Agent 的工作方式，也可能引导它使用文件或工具。添加第三方 Skill 前，应查看：

- 来源是否可信。
- 说明中要求访问哪些内容。
- 是否包含不必要的命令或外部连接要求。
- 资源中是否夹带账号、密钥或个人信息。

Skill 本身不会绕过正式的文件、命令或 MCP 审批。

## 列表中找不到 Skill

按以下顺序检查：

1. Skill 是否放在正确的系统级或项目级目录。
2. 目录内是否存在有效的 `SKILL.md`。
3. 当前会话是否绑定了预期项目。
4. 是否有更高优先级的同名 Skill 覆盖了它。
5. 更高层同名目录是否存在格式错误，因安全阻断而遮住了低层版本。

系统级继承是固定规则，当前没有“关闭继承”或“停用某个 Skill”的目录配置，不要按旧版 `keydex.json` 排查。修正后重新打开选择器；仍未更新时，重新进入会话或使用刷新入口。

## 内容错误

名称、描述、Frontmatter 或文件结构有问题时，Skill 可能无法显示或无法正确匹配。根据界面中的诊断提示修正对应文件，不要通过复制多个同名目录反复尝试。Skill 目录中的符号链接、目录联接或越界资源会被拒绝。

## 更新规则

- 内置 Skill 由应用发布方维护。应用更新时，以发布版本覆盖，确保所有用户获得一致内容。
- 系统级和项目级 Skill 属于用户或项目，应用更新不应覆盖。
- 想定制内置 Skill 时，在系统级或项目级创建同名版本，通过优先级覆盖。

## 定制后的维护

高优先级定制会持续遮盖后续内置更新。应用升级后，如果希望采用新版内置内容，应比较自己的定制，移除或改名同名覆盖项，再确认选择列表中的来源。
