# 消息、工具过程、审批与错误提示

## 阅读一次完整回复

一轮任务可能包含文字说明、操作过程、审批请求、A2UI 交互和最终答复。判断任务是否完成时，以最终可见结果为准，不要只看某一步显示“已调用”。

工具过程用于说明 Agent 正在查看或操作什么。普通产品使用问题不需要理解内部名称，但应关注：

- 是否仍在运行。
- 是否正在等待用户批准。
- 是否需要填写表单或做出选择。
- 是否已经失败并给出恢复入口。

悬停一轮 Agent 回复中的任意内容时，操作区会显示在该轮末尾。“复制”会合并这一轮的全部 Agent 文本片段，而不是只复制鼠标旁的一小段；工具入参、输出和错误卡片仍使用各自的复制入口。

## 处理审批

命令执行、文件访问或 MCP 操作可能需要审批。批准前检查操作对象、影响范围和是否需要长期信任。

- 只信任本次：适合不确定或一次性的操作。
- 会话内信任：适合同一会话中反复执行的明确操作。
- 持久信任：只用于来源可靠、范围清楚的长期规则。

拒绝审批不会删除会话，可以补充要求，让 Agent 改用更小的范围或只做分析。

## 处理错误

错误卡片优先显示可理解的业务提示，还可能包含错误码和 HTTP 状态。需要进一步定位时展开“错误详情”；向他人求助时使用“复制错误”，它会连同当前任务、回合和消息位置等必要上下文复制，并对常见敏感值做清洗和长度限制。

看到失败提示时，先读提示中的对象和建议操作：

1. 如果支持重试，确认前置条件后重试。
2. 如果是权限问题，缩小访问范围或重新审批。
3. 如果是文件变化，重新打开文件并确认最新内容。
4. 如果是服务或模型问题，检查对应设置和连接状态。

不要连续重复同一高影响操作。重复失败时，把“复制错误”的内容、当前页面、刚执行的操作和 Keydex 版本一起提供；粘贴前仍应人工检查路径、业务数据和凭据是否适合分享。

错误详情在实时回复和重新打开的历史记录中应保持一致。若刷新前后详情丢失或只剩笼统提示，保留截图并报告为显示问题，不要靠反复触发高风险操作重新制造错误。

## 结果不一致

工具步骤显示成功但页面结果未变化时，先确认目标文件、会话或面板是否正确，再进行一次最小刷新。不要直接重复执行可能产生重复数据的操作。
