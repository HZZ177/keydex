# `keydex.md` 生成模板与合并规则

## 目标文件

- 项目级：`<project>/.keydex/keydex.md`
- 系统级（仅当用户明确迁移跨项目偏好）：`~/.keydex/keydex.md`

项目 `/init-keydex` 默认只写项目级。

## 推荐结构

```markdown
# 项目工作说明

## 项目与范围
- 本仓库负责什么
- 明确不在范围内的部分

## 目录地图
- `path/`：职责（只写关键目录）

## 环境与命令
- 安装/启动
- 构建
- Lint
- 测试（含如何跑单个测试）

## 工程约定
- 代码与变更约束（只写项目特有约定）

## 验证与完成标准
- 改动后如何验证
- 交付时应说明什么

## 风险与禁止事项
- 不可做的操作、敏感边界

## 迁移来源（可选，宜短）
- 已吸收：CLAUDE.md / AGENTS.md / ...
- 未迁移：hooks/settings（需在 Keydex 设置重配）
```

## 写入质量规则

1. 短小、可执行，优先写代码里不显然的信息
2. 不写通用鸡汤（“写好测试”“别提交密钥”除非项目有特殊流程）
3. 不枚举全部文件树
4. 命令必须来自真实脚本/文档/CI，不编造
5. 总大小 ≤ 32 KiB
6. 长流程不要塞这里，改为：`详细流程见 skill: /xxx`

## 合并优先级

高 → 低：

1. 用户在 A2UI 中的明确选择
2. 现有 `.keydex/keydex.md`
3. 项目共享记忆（`CLAUDE.md` / `AGENTS.md`）
4. README / 包管理脚本 / CI 中的事实
5. 代码结构推断
6. 禁止填充虚构章节

## 从 CLAUDE/AGENTS 吸收时

保留：

- 架构要点
- 真实命令
- 模块边界
- 项目特有约定
- 验证要求

删除或改写：

- Claude/Codex 专属权限语法
- hook 配置
- `@import` 路径（展开必要正文或改为 skill references）
- 本机绝对路径
- 密钥与隐私
- 与 Keydex 能力冲突的指令（如要求绕过审批）

## 已有文件策略

| 策略 | 行为 |
|---|---|
| create | 直接按模板新建 |
| merge | 以现有结构为骨，补全缺失命令/架构/迁移要点，去重 |
| overwrite | 仅用户明确确认后全量重写 |
| skip_md | 不改 `keydex.md` |

## 完成后自检

- [ ] 有项目范围
- [ ] 有至少一组真实命令（若仓库可识别）
- [ ] 有验证方式
- [ ] 无密钥
- [ ] 无大段可发现目录枚举
- [ ] ≤ 32 KiB
